B2B Data Sharingc back to ToC or Class ToC

IRI: https://w3id.org/def/dataact#B2BDataSharing

Chapter III (Article 8) - Business-to-business data sharing, must be authorized by user per Article 8(4)
Data sharing between businesses where a data recipient requests access from a data holder. MANDATORY REQUIREMENTS: (1) User authorization (Article 8(4)), (2) FRAND terms (Article 8(1)), (3) Non-discrimination (Article 8(3)). Data holder may refuse ONLY if data contains trade secrets (Article 8(6)).
Example
Aftermarket repair service (autoRepair) requests robot diagnostic data from RobotManufacturer with user authorization. Violation example: RobotManufacturer refuses without trade secret justification (Article 8(6) violation).
has super-classes
Data Sharing c

B2C Data Sharingc back to ToC or Class ToC

IRI: https://w3id.org/def/dataact#B2CDataSharing

Chapter II (Articles 3-7) - Business-to-Consumer data sharing, data holder makes data available to user
Data sharing from data holder to user (consumer or enterprise) who owns, rents, or leases the connected product that generated the data. Data holders must make data available without undue delay, free of charge, and in machine-readable format (Article 4).
Example
Consumer (Charlie) requests health data from smart watch manufacturer - manufacturer must provide the data. Factory owner requests sensor data from IoT device manufacturer. Violation example: watchManufacturer fails to provide charlieHealthData despite user request.
has super-classes
Data Sharing c

B2G Data Sharingc back to ToC or Class ToC

IRI: https://w3id.org/def/dataact#B2GDataSharing

Chapter V (Articles 14-22) - Business to Government data sharing for exceptional need
Data sharing from data holder to public sector body based on exceptional need (Article 15) such as public emergency, official statistics, or regulatory function. CRITICAL RESTRICTION: Article 19(2)(a) PROHIBITS using received data to develop products or services competing with private sector.
Example
HealthAuthority requests health data for public emergency response - ALLOWED. HealthAuthority uses same data to develop competing health app - PROHIBITED (Article 19(2)(a) violation). Statistical office requests industrial data for GDP calculation - ALLOWED.
has super-classes
Data Sharing c

Data Provisionc back to ToC or Class ToC

FRAND Conditionc back to ToC or Class ToC

IRI: https://w3id.org/def/dataact#FRANDCondition

Article 8-12 - FRAND conditions
Fair, Reasonable, And Non-Discriminatory conditions mandatory for B2B data sharing agreements (Article 8(1)). Ensures data recipients can access data under balanced terms. COMPLIANCE METRICS: isFair, isReasonable, isNonDiscriminatory properties must all be true. Violations render contractual terms unenforceable (Article 13).
Example
frand189 (all properties true) = compliant B2B agreement. frand247 (isFair=false, isNonDiscriminatory=false) = Article 8(1) violation, terms unenforceable. compensationAmount must be reasonable. Non-discrimination: comparable recipients get comparable terms (Article 8(3)).
is in range of
subject to FRAND op
